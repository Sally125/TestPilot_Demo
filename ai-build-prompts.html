# 项目背景

本项目是一个面向 QA 工程师的开源 AI 自动化测试工作台。QA 工程师通过输入需求文档（飞书链接 / 纯文本 / 代码片段），由 AI 引擎自动分析需求并生成结构化测试用例，再通过 Node.js 执行层自动化运行用例，最终生成测试报告。项目采用双服务架构：FastAPI 作为调度层负责业务逻辑与 AI 编排，Node.js 作为执行层负责测试用例的实际运行。

# 当前模块目标

搭建整个项目的 monorepo 基础架构，使 `pnpm install` 和 `pip install -e .` 后前后端均可独立启动，数据库迁移可成功创建全部 6 张核心表，JWT 认证流程可跑通。本模块不实现具体业务页面，只产出可运行的项目骨架。

# 技术栈约束

- 包管理：pnpm 9+（workspace 模式）
- 前端：Vite 5 + React 18 + TypeScript 5
- 后端：FastAPI 0.110+ + Python 3.11+ + SQLModel 0.0.16+
- 数据库：PostgreSQL 15+，需启用 pgvector 扩展（用于后续 AI 向量检索）
- 缓存/队列：Redis 7+
- 认证：PyJWT 2.8+ + passlib[bcrypt]
- 迁移工具：Alembic
- 配置管理：pydantic-settings（后端）/ Vite env（前端）

# API 路由定义

所有路由统一前缀 `/api/v1`。

## 认证模块 /api/v1/auth
- `POST /register` — 注册
  - 请求体: `{ email: str, password: str(>=8), name: str }`
  - 响应: `{ id, email, name, role, created_at }`
  - 校验: email 格式 + 唯一性；password >= 8 字符
- `POST /login` — 登录
  - 请求体: `{ email, password }`
  - 响应: `{ access_token, refresh_token, token_type: "bearer", user: {...} }`
- `POST /refresh` — 刷新 token
  - 请求体: `{ refresh_token }`
  - 响应: `{ access_token, token_type: "bearer" }`
- `GET /me` — 获取当前用户（需 Bearer token）
  - 响应: `{ id, email, name, avatar_url, role, last_login_at }`

## 项目模块 /api/v1/projects
- `GET /` — 项目列表（分页，page/page_size）
- `POST /` — 创建项目
- `GET /{id}` — 项目详情
- `PUT /{id}` — 更新项目
- `DELETE /{id}` — 删除项目（软删除，status -> archived）

## 用户模块 /api/v1/users
- `GET /me` — 当前用户信息
- `PUT /me` — 更新个人信息
- `PUT /me/password` — 修改密码（需验证旧密码）

# 安全模块实现规格（core/security.py）

- 密码哈希: passlib CryptContext(schemes=["bcrypt"], bcrypt__rounds=12)
- create_access_token(subject: str) -> str: JWT payload 包含 sub(user_id), exp(30min), iat, type="access"
- create_refresh_token(subject: str) -> str: exp(7天), type="refresh"
- verify_token(token: str) -> dict: 验证签名+过期，返回 payload
- get_current_user 依赖: 从 Authorization header 提取 Bearer token -> 验证 -> 查库返回 user 对象，无效则抛 401

# 数据库配置（core/database.py）

- engine = create_async_engine(settings.DATABASE_URL, pool_size=20, max_overflow=10, echo=False)
- DATABASE_URL 格式: postgresql+asyncpg://user:pass@host:5432/dbname
- get_session() async generator: yield AsyncSession
- 启动时执行 `CREATE EXTENSION IF NOT EXISTS vector`（pgvector）

# 配置管理（core/config.py）

使用 pydantic-settings BaseSettings，从 .env 读取:

```python
class Settings(BaseSettings):

    DEBUG: bool = True

    JWT_ALGORITHM: str = "HS256"

    REFRESH_TOKEN_EXPIRE_DAYS: int = 7

    # CORS
    CORS_ORIGINS: list[str] = ["http://localhost:5173"]

    model_config = SettingsConfigDict(env_file=".env", case_sensitive=True)
```

# 前端 axios 封装（api/request.ts）

- 创建 axios 实例，baseURL = import.meta.env.VITE_API_BASE_URL || "/api/v1"
- 请求拦截器: 自动附加 Authorization: Bearer {access_token}（从 localStorage 读取）
- 响应拦截器:
  - 200: 返回 response.data
  - 401: 清除 token，跳转 /login
  - 403: 提示无权限
  - 500: 提示服务器错误
  - 网络错误: 提示网络异常

# docker-compose.yml

提供 PostgreSQL + Redis 两个服务:
- postgres: image postgres:15-alpine，端口 5432，环境变量 POSTGRES_USER/PASSWORD/DB，挂载 init.sql（创建 pgvector 扩展）
- redis: image redis:7-alpine，端口 6379

# 验收标准

1. `pnpm install` 在根目录执行成功，无 peer dependency 警告
2. `pnpm --filter frontend dev` 启动 Vite 开发服务器，访问 http://localhost:5173 显示空白 React 页面
3. 后端 `cd backend && pip install -e . && uvicorn app.main:app --reload` 启动成功，访问 http://localhost:8000/docs 显示 Swagger 文档
4. `docker-compose up -d` 启动 PostgreSQL + Redis
5. `alembic upgrade head` 成功创建全部 6 张表 + pgvector 扩展
6. POST /api/v1/auth/register 可成功注册用户，密码在数据库中为 bcrypt 哈希
7. POST /api/v1/auth/login 返回 access_token 和 refresh_token
8. GET /api/v1/auth/me 携带 Bearer token 可获取用户信息，无 token 返回 401
9. POST /api/v1/auth/refresh 可用 refresh_token 换取新 access_token
10. 前端 axios 实例能成功调用后端 /api/v1/auth/me（通过 Vite proxy 转发）

# 注意事项

- 密码绝不明文存储或返回，所有用户响应 schema 排除 password_hash 字段
- JWT_SECRET_KEY 必须从环境变量读取，代码中不得硬编码默认值
- 数据库使用 asyncpg 异步驱动，所有 DB 操作使用 async/await
- Alembic 的 env.py 需配置为读取 SQLModel.metadata，target_metadata = SQLModel.metadata
- PostgreSQL 连接池 pool_size=20，max_overflow=10，避免连接泄漏
- CORS 仅允许前端开发端口，生产环境需配置实际域名
- docker-compose 的 PostgreSQL 数据卷需持久化，避免容器删除后数据丢失
- .env 文件必须加入 .gitignore，仅提交 .env.example
- pgvector 扩展需在数据库初始化时通过 init.sql 或 Alembic 迁移创建
- 所有 API 路由添加 OpenAPI tags 分组（auth / projects / users）便于 Swagger 文档组织

本项目是一个面向 QA 工程师的开源 AI 自动化测试工作台。前端基于 Vite + React 18 + TypeScript + Ant Design 5 构建，视觉设计遵循飞书设计语言。本模块在模块 1 搭建的项目骨架基础上，实现 SaaS 后台全局布局框架，为后续所有业务页面提供统一的布局容器、导航体系和路由入口。

# 当前模块目标

实现一个完整的 SaaS 后台布局：顶部固定 Header（56px）+ 左侧可折叠导航栏（240px/64px）+ 右侧内容滚动区。配置 React Router v6 路由表，定制 Ant Design 5 飞书主题色，使用 Zustand 管理认证状态与布局状态，并实现登录页使认证流程可端到端跑通。

# 技术栈约束

- 路由：react-router-dom 6.22+
- UI 组件库：antd 5.15+
- 图标：@ant-design/icons 5.3+
- 状态管理：zustand 4.5+
- HTTP 客户端：axios（已在模块 1 封装 api/request.ts）
- 样式方案：CSS Modules 或 antd 内置样式 + 少量全局 CSS 变量

# Zustand Store 定义

## useAuthStore（stores/auth.ts）

```typescript
interface AuthState {
  token: string | null;
  refreshToken: string | null;
  user: User | null;
  setAuth: (data: { access_token: string; refresh_token: string; user: User }) => void;
  setUser: (user: User) => void;
  logout: () => void;
}
```

- 初始值从 localStorage 读取 token（持久化）
- setAuth: 写入 state + localStorage
- logout: 清空 state + localStorage，移除 Authorization header
- User 类型: { id, email, name, avatar_url, role }

## useLayoutStore（stores/layout.ts）

```typescript
interface LayoutState {
  collapsed: boolean;
  toggleCollapsed: () => void;
  setCollapsed: (val: boolean) => void;
}
```

- collapsed 初始值从 localStorage 读取，默认 false
- 切换时同步写入 localStorage（key: "layout_collapsed"）

# 验收标准

1. 访问 http://localhost:5173 自动重定向到 /login
2. 登录页表单校验正常（空值、邮箱格式、密码长度）
3. 输入正确凭据登录成功，跳转到首页 /，token 写入 localStorage
4. 登录后 Header 显示用户名和头像，Sider 显示 6 项导航菜单
5. 点击折叠按钮，Sider 从 240px 平滑过渡到 64px，仅显示图标，刷新后保持折叠状态
6. 点击各菜单项正确路由到对应页面（占位页显示页面名称即可）
7. 直接访问未登录的路由（如 /requirements），无 token 时重定向到 /login
8. 退出登录后清除 token，重定向到 /login
9. Ant Design 组件颜色为主色 #1456F0，按钮圆角 6px，卡片圆角 12px
10. 页面在 1440px 和 1920px 宽度下布局正常，无溢出

# 注意事项

- 路由懒加载使用 React.lazy + Suspense，fallback 使用 antd Spin 居中
- localStorage 存取 token 时注意 JSON 序列化，token 过期后响应拦截器自动清除并跳转登录
- Sider 折叠状态持久化到 localStorage，避免刷新后重置
- Menu 的 selectedKeys 需与当前路由 pathname 同步（useLocation 获取）
- 嵌套路由（如 /requirements/:id/analysis）需在 MainLayout 的 children 中正确定义
- antd locale 需引入 zhCN（dayjs 也要配置中文 locale）
- 404 页面需独立配置，不使用 MainLayout
- Content 区域高度计算: calc(100vh - 56px)，内部 overflow-y: auto
- 折叠按钮在 Sider 内底部或 Header 内左侧均可，但需保证 56px Header 内可点击
- 所有占位页面导出默认组件，内容为居中显示页面名称，方便后续模块替换

## GET /api/v1/dashboard/stats
返回统计卡片数据:
```json
{
  "pass_rate": 87.5,
  "pass_rate_change": 2.3,
  "failed_cases": 12,
  "failed_cases_change": -3,
  "total_cases": 156,
  "total_cases_change": 8,
  "weekly_new_cases": 23,
  "weekly_new_cases_change": 5
}
```

## GET /api/v1/dashboard/pass-rate-trend?days=7
返回趋势数据:
```json
{
  "dates": ["07-06", "07-07", "07-08", "07-09", "07-10", "07-11", "07-12"],
  "pass_rates": [82.1, 85.3, 84.7, 87.2, 86.5, 88.1, 87.5]
}
```
- days 参数: 7 或 30
- 后端聚合逻辑: 按日期分组 execution_records，计算每日 passed / total * 100

## GET /api/v1/dashboard/pending-reviews?limit=5

# 前端 API 封装（api/dashboard.ts）

```typescript
import request from "./request";

export const getDashboardStats = () =>
  request.get("/dashboard/stats");

export const getPassRateTrend = (days: number) =>
  request.get<{ dates: string[]; pass_rates: number[] }>(
    "/dashboard/pass-rate-trend", { params: { days } }
  );

export const getPendingReviews = (limit = 5) =>
  request.get("/dashboard/pending-reviews", { params: { limit } });

export const getRecentExecutions = (limit = 10) =>
  request.get("/dashboard/recent-executions", { params: { limit } });
```

# Dashboard 页面组件（pages/Dashboard.tsx）

```typescript
function Dashboard() {
  const [stats, setStats] = useState(null);
  const [trendDays, setTrendDays] = useState(7);
  const [trendData, setTrendData] = useState<{dates:string[]; pass_rates:number[]}>();
  const [pendingReviews, setPendingReviews] = useState([]);
  const [recentExecutions, setRecentExecutions] = useState([]);
  const [loading, setLoading] = useState(true);

  // 页面加载时并行请求所有数据
  useEffect(() => {
    Promise.all([
      getDashboardStats(),
      getPassRateTrend(7),
      getPendingReviews(),
      getRecentExecutions()
    ]).then(([s, t, p, r]) => {
      setStats(s); setTrendData(t); setPendingReviews(p); setRecentExecutions(r);
    }).finally(() => setLoading(false));
  }, []);

  // 切换 7天/30天 时重新请求趋势数据
  useEffect(() => {
    getPassRateTrend(trendDays).then(setTrendData);
  }, [trendDays]);

  // loading 时显示 Skeleton
  // 渲染: 页面标题 + Row/Col 栅格布局
}
```

# 注意事项

- ECharts 实例需在组件卸载时销毁（echarts-for-react 自动处理，但需确认无内存泄漏）
- 趋势图 resize: 窗口大小变化时图表需自适应（使用 ResizeObserver 或 echarts resize 事件）
- 4 个 API 并行请求使用 Promise.all，单个失败不影响其他数据展示（改为 Promise.allSettled）
- 耗时格式化函数需处理 null 值（未完成的执行记录 duration_ms 为 null，显示 "—"）
- 相对时间显示使用 dayjs.relativeTime 插件，需在项目入口注册
- 表格列的 test_case_title 点击跳转需使用 react-router 的 useNavigate
- Segmented 切换时如网络慢，需显示 loading 态防止重复点击
- 统计卡片的环比变化箭头方向需根据指标语义判断（通过率↑为好，失败数↑为坏）
- 响应式断点: <992px 时统计卡片改为 Col span=12（2列），中间区域改为上下堆叠
- 所有金额/数值显示需考虑千分位分隔（用 toLocaleString）

# 项目背景

本项目是一个面向 QA 工程师的开源 AI 自动化测试工作台。QA 工程师通过输入需求文档（飞书链接 / 纯文本 / 代码片段），由 AI 引擎自动分析需求并生成测试用例。前端基于 Vite + React 18 + TypeScript + Ant Design 5，后端基于 FastAPI + Python + SQLModel，视觉设计遵循飞书设计语言（品牌主色 #1456F0，卡片圆角 12px）。

模块 1 完成了数据库设计（requirements 表含 source_type / source_content / feishu_url / status 等字段），模块 2 实现了全局布局与路由（/requirements 路由已注册）。本模块实现需求管理页，是用户输入需求、查看历史需求的核心入口。

# 当前模块目标

实现需求管理页完整功能：上半部分为新建需求区域（Tab 切换三种输入模式 + 提交按钮），下半部分为历史需求列表表格。飞书链接模式支持 URL 校验与后端拉取文档预览，需求文本模式提供 Markdown 富文本编辑器，代码文本模式提供语法高亮代码编辑器。提交后创建需求记录，状态为 "待分析"，可跳转到模块 5 的 AI 分析页。

# Tab 输入模式详细规格

## Tab 1: 飞书链接（source_type = "feishu"）

布局:
```
飞书文档链接: [https://xxx.feishu.cn/docs/_______________] [校验并预览]
                                                           ↑ Button primary

┌─ 文档预览 ────────────────────────────────────┐
│  (校验成功后显示文档标题 + 内容预览)              │
│  文档标题: XXX需求文档                          │
│  ─────────────────────────────────            │
│  文档内容 (Markdown 渲染，只读)...              │
│                                               │
│  (校验失败显示错误提示: 红色 Alert)              │
│  (未校验显示提示: 灰色 "粘贴飞书文档链接后点击校验") │
└───────────────────────────────────────────────┘
```

飞书 URL 校验规则（前端）:
- 正则: `^https://[\w-]+\.feishu\.cn/(docs|wiki|docx)/([a-zA-Z0-9]+)`
- 匹配 https://xxx.feishu.cn/docs/XXXXX 或 https://xxx.feishu.cn/wiki/XXXXX
- 不匹配时 Input 显示红色错误态 + 提示 "请输入有效的飞书文档链接"

校验并预览交互:
1. 点击 "校验并预览" 按钮
2. 按钮显示 loading
3. 调用后端 POST /api/v1/requirements/feishu/preview，传 { url }
4. 成功: 预览区显示文档标题 + Markdown 渲染内容，提取 doc_token 存入表单状态
5. 失败: 预览区显示 antd Alert (type=error)，提示后端返回的错误信息
6. 常见错误: 链接无效 / 文档无访问权限 / 飞书 App 未授权
